# Database Design

Use this skill when creating or changing relational/non-relational data models.

## Focus

- Schema design aligned to query patterns
- Migration safety and rollback strategy
- Data integrity constraints and indexing

## Workflow

1. Model entities/relations and expected access patterns.
2. Add constraints and indexes for correctness and performance.
3. Write reversible migration steps when possible.
4. Validate query plans on representative data.
